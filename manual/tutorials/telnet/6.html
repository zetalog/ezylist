Telnet Tutorial - Debugging CGI Scripts

Don't you hate it when you try to run a CGI script and you get a big ugly "500 Internal Server Error?" Here's a little trick which will help you find out what's wrong with the CGI script. All you have to do, it telnet into your account, change directories to the directory your CGI script is in, and type ./script.cgi. The ./ means run this program, and script.cgi tells the server which program to run.

Your server will then try to run the CGI script. If anything goes wrong, it will give off an explanation of the error which occurred. It will print out an error on your telnet screen, like Permission Denied, which means you have to chmod that file to 755.
